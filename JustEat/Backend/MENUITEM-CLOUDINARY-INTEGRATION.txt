  - image: [FILE] (required, JPEG/PNG/WebP/GIF, max 5MB)

───────────────────────────────────────────────────────────────────

🔄 CLOUDINARY FLOW FOR MENU ITEMS:

1. Owner submits menu item with image file
2. MenuItemController receives multipart form data
3. Validates all parameters (name, price, description, etc.)
4. Passes to MenuItemService with image file
5. MenuItemServiceImpl uploads image to Cloudinary folder: "justeat/menu-items"
6. Cloudinary returns:
   - secure_url → saved as imageUrl
   - public_id → saved as imagePublicId
7. MenuItem created in database with Cloudinary URLs
8. Response returned to frontend

───────────────────────────────────────────────────────────────────

🎯 VALIDATION RULES:

Field Validations:
- name: Required, 2-100 characters
- price: Required, must be > 0.0
- description: Required, max 500 characters
- cuisineType: Required enum (INDIAN, CHINESE, ITALIAN, etc.)
- dietaryRestriction: Required enum (VEGETARIAN, NON_VEGETARIAN, VEGAN)
- isSpecial: Optional, boolean, defaults to false
- image: Required file

Image Validations (from CloudinaryService):
- File Type: JPEG, PNG, WebP, or GIF only
- File Size: Maximum 5MB
- File Required: Cannot be empty

const addMenuItem = async (restaurantId, formData) => {
  const data = new FormData();
  data.append('name', formData.name);
  data.append('price', formData.price);
  data.append('description', formData.description);
  data.append('cuisineType', formData.cuisineType);
  data.append('dietaryRestriction', formData.dietaryRestriction);
  data.append('isSpecial', formData.isSpecial || false);
  data.append('image', formData.imageFile); // File object

  return api.post(`/restaurants/${restaurantId}/menu`, data, {
    headers: {
      'Content-Type': 'multipart/form-data'
    }
  });
};
